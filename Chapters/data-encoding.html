Chapter 7: Data Encoding

How text, images, and sound become 0s and 1s! 🎨🎵

Encoding Text (ASCII)

ASCII (American Standard Code for Information Interchange) is a system where each letter, number, and symbol gets a unique number from 0 to 127.

🔤 Examples:
'A' = 65 → binary 01000001
'a' = 97 → binary 01100001
'0' = 48 → binary 00110000
' ' (space) = 32 → binary 00100000

ASCII uses 7 bits (128 characters). But since we often use 8-bit bytes, we get Extended ASCII with 256 characters.

Unicode & UTF-8

ASCII only works for English! 😞 For Urdu, Chinese, emojis 😊, etc., we use Unicode (21-bit codes, millions of characters) encoded in UTF-8.

Encoding Numbers

Positive Integers

Plain binary: 5 → 101, 10 → 1010

Negative Integers: 2's Complement

How to store negative numbers? Use 2's Complement:

To find -5 (using 8 bits):

1. Start with +5: 00000101

2. Flip all bits: 11111010

3. Add 1: 11111011

✅ So -5 = 11111011

Decimal Numbers: Floating Point

For numbers like 3.14 or 0.5, computers use Floating Point Notation with 3 parts:

  • Sign Bit (1 bit): + or −
  • Exponent (8 bits): moves decimal point
  • Mantissa (23 bits): the actual digits

Encoding Images (Pixels)

An image is made of tiny dots called pixels 🎨. Each pixel has a color.

Try it: Click pixels to draw!

RGB Color System

Every color = mix of Red + Green + Blue.

🎨 How it works:
Each color value is 0-255 (1 byte).
So 1 pixel = 3 bytes (24 bits) for true color!
Example: Pure red = (255, 0, 0) = #FF0000

💡 Try the RGB Color Mixer on the home page!

Encoding Sound

Sound is a wave 🌊. To store it, we take samples — measuring the wave's height thousands of times per second.

📊 Sampling rates:
📞 Phone: 8,000 samples/sec (good enough for voice)
🎵 CD Music: 44,100 samples/sec (studio quality)

MIDI

MIDI (Musical Instrument Digital Interface) is different — instead of recording sound, it records instructions (play note C, hold 2 seconds). Like sheet music! 🎼

Data Compression

Files can be BIG 📦. Compression makes them smaller!

Lossless

No data lost. You can recover the original exactly.

Examples: ZIP, PNG, GIF

Methods: Run-Length Encoding, Huffman, LZW

Lossy

Some data thrown away (things eyes/ears don't notice). Much smaller files!

Examples: JPEG, MP3, MPEG (video)

Summary

✅ Text: ASCII (7-bit), Unicode (supports all languages)

✅ Numbers: Binary (+), 2's Complement (-), Floating Point (decimals)

✅ Images: Pixels + RGB (3 bytes per pixel)

✅ Sound: Sampling (8000 phone / 44100 CD)

✅ Compression: Lossless (ZIP) or Lossy (JPEG, MP3)
