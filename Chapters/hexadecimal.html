Chapter 5: Hexadecimal

A shorter way to write binary — used everywhere in programming! 🎨

Why Hexadecimal?

Imagine writing: 11111111 00001010 10110011... too long, right? 😵

Hexadecimal (or "hex") is a shorthand that makes long binary much shorter and easier to read!

🎯 Key idea: One hex digit = 4 bits. So 8 bits (1 byte) = just 2 hex digits!

Binary → Hex (Super Easy!)

Steps:

  1. Split binary into groups of 4 (from right)
  2. Convert each group to its hex symbol
  3. Done! 🎉

Example: Convert 11011010 to Hex

Step 1: Split into 4-bit groups
1101 1010

Step 2: Convert each
1101 = D
1010 = A

Step 3: Combine
Answer: DA (or 0xDA)

Where You See Hex

🎨

Web Colors

#FF5733 in CSS = a color!
FF=Red, 57=Green, 33=Blue

💾

Memory Addresses

0x7FFE2A00 — where data lives in RAM

🆔

MAC Addresses

AA:BB:CC:DD:EE:FF — your device's unique ID

Summary

✅ Hex uses 16 symbols: 0-9, A-F

✅ 1 hex digit = 4 bits

✅ 1 byte = 2 hex digits

✅ Used in colors, memory, MAC addresses

✅ Usually written with 0x prefix (0xFF)
