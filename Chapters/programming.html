Chapter 12: Programming Languages

How humans talk to computers! 💬🤖

What is a Programming Language?

Computers only understand binary (0s and 1s). But writing binary is terrible for humans! 😫

So we invented programming languages — a way to write instructions in almost English, which gets translated to binary.

🌉 Think of it like translation:
You write Python (like English) → Translator (compiler/interpreter) → Computer binary → Computer does the task!

Generations of Languages

1st

Machine Language

Pure binary: 10110000 01100001

The ONLY language computer directly understands. But impossible for humans! 😵

2nd

Assembly Language

Uses short codes instead of binary: MOV AL, 61h

Still hard, but better than pure binary. Needs an assembler to convert.

3rd

High-Level Languages

Look almost like English! Examples: Python, Java, C++, JavaScript, C#

print("Hello, World!")

Needs a compiler or interpreter to translate to machine language.

Programming Paradigms

Different STYLES of writing code:

👨‍🍳

Imperative

Give step-by-step commands. "Do this, then this, then this." (C, Pascal)

📝

Declarative

Describe WHAT you want, not how. (SQL: "SELECT all users")

🧮

Functional

Everything is a function. Pure, no side-effects. (Haskell, Lisp)

📦

Object-Oriented

Organize code as "objects" with data & actions. (Java, Python, C++)

Object-Oriented Programming (OOP)

OOP is the MOST POPULAR modern paradigm. Key ideas:

🏗️ Class = blueprint (like a "Car" design)
🚗 Object = actual thing made from blueprint (your specific car)
🎨 Properties = data (color, speed)
🔧 Methods = actions (drive, stop, honk)

class Car {

  color = "red";

  speed = 0;

  function drive() { speed += 10; }

}

myCar = new Car(); // An object!

myCar.drive(); // Uses method

OOP Benefits

  • ♻️ Reusability: Write once, use many times
  • 🔒 Information Hiding: Keep internal data safe
  • 🧩 Modularity: Break code into manageable pieces

Summary

✅ Generations: Machine → Assembly → High-Level

✅ Compiler = translates whole program at once

✅ Interpreter = translates line by line

✅ Paradigms: Imperative, Declarative, Functional, OOP

✅ OOP uses Classes, Objects, Methods
