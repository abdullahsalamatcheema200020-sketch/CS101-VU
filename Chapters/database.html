Chapter 14: Database Systems

How to organize and manage HUGE amounts of data! 📊

What is a Database?

A database is an organized collection of data. A DBMS (Database Management System) is software to manage it.

📚 Think of a library:
Without organization → chaos! 😱
Library uses: categories, shelves, catalog system → easy to find any book! 📖
A database does the same for DATA.

Flat Files vs Databases

  • 📄 Flat File (like Excel): Simple, but hard for huge data
  • 💾 Database: Professional, fast, handles millions of records

The Relational Model

Data is stored in tables (rows and columns), like Excel but much more powerful!

Example: Student Table

StudentIDNameProgramCGPA
BC001Ali KhanBSCS3.8
BC002Sara AhmedBSCS3.9
BC003Usman AliBSIT3.5
📝 Terms:
• Table = relation
• Row = tuple / record (one student)
• Column = attribute (one field like Name)
• Primary Key = unique ID (StudentID)

Relational Operations

SELECT

Pick rows that match a condition.
WHERE CGPA > 3.5

PROJECT

Pick specific columns.
SELECT Name, CGPA

JOIN

Combine two related tables into one result.

SQL — The Database Language

SQL (Structured Query Language) is how we talk to databases:

SELECT Name, CGPA

FROM Students

WHERE CGPA > 3.5

ORDER BY CGPA DESC;

This returns students with CGPA above 3.5, sorted highest first!

Database Integrity

Databases must stay accurate even when many users use them at once!

  • ✅ Commit: Save changes permanently
  • ↩️ Rollback: Undo if something goes wrong
  • 🔒 Locking: Prevent two users editing same thing
  • 🔑 Hashing: Fast data lookup

Summary

✅ Database = organized data, managed by DBMS

✅ Data stored in tables (rows & columns)

✅ Operations: SELECT, PROJECT, JOIN

✅ SQL is the standard database language

✅ Examples: MySQL, PostgreSQL, MS Access, Oracle
