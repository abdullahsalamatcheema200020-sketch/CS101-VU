Chapter 13: Software Engineering

How professional software is built — like building a house! 🏗️

What is Software Engineering?

Software Engineering is the discipline of building large, reliable software in an organized way.

🏠 Like building a house:
You don't just start laying bricks! You need a plan, architect, materials, workers, testing, and maintenance.
Software is the SAME — we follow a process called the Software Life Cycle.

The Software Life Cycle

4 main phases every software goes through:

1. Requirements

What should the software do? Talk to clients, understand needs. 📋

2. Design

Plan the structure. Draw diagrams, choose technologies. 🎨

3. Implementation

Write the actual code! Programmers build the software. 💻

4. Testing

Find bugs 🐛, fix them, make sure software works correctly!

➕ After release, there's also Maintenance — fixing bugs and adding features based on user feedback.

Development Models

💧

Waterfall Model

Each phase completes before next begins. Like a waterfall — goes one direction.

✅ Simple, clear • ❌ No going back, risky for big projects

📈

Incremental Model

Build software in small pieces. Add features step by step. 🧱

🔄

Iterative Model

Build a basic version, improve it again and again. 🔁

🚀

Agile (XP)

Most popular today! Small teams, frequent releases, quick feedback. Extreme Programming (XP) is one style.

✅ Flexible, fast, adapts to changes

Design Principles

  • 🧩 Modularity: Break big software into small, manageable pieces
  • 🔗 Coupling: How much modules depend on each other (want LOW)
  • 💎 Cohesion: How focused a module is (want HIGH)
  • 🔐 Information Hiding: Keep internal details private
💡 Good design = Low Coupling + High Cohesion

Summary

✅ Life Cycle: Requirements → Design → Implementation → Testing

✅ Models: Waterfall, Incremental, Iterative, Agile

✅ Agile is most popular today (flexible, fast)

✅ Good design: Low Coupling + High Cohesion
