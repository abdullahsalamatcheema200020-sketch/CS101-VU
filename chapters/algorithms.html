Chapter 11: Algorithms

Step-by-step solutions to solve any problem! 🧩

What is an Algorithm?

An algorithm is an ordered set of clear, executable steps that solves a problem in finite time.

🍳 Real-life example: Making a cup of tea!
1. Boil water
2. Add tea bag
3. Wait 3 minutes
4. Add milk & sugar
5. Stir and enjoy! ☕
That's an algorithm! Every step is clear and leads to a result.

Properties of a Good Algorithm

  • ✅ Unambiguous: Each step is clear
  • ✅ Executable: Each step is doable
  • ✅ Terminating: Must end eventually (not run forever!)
  • ✅ Ordered: Steps in correct sequence

Simple Algorithm Example

Problem: Convert Kilometers to Meters

Step 1: Get input — number of kilometers (KM)

Step 2: Multiply KM by 1000

Step 3: Show the result in meters

✨ Simple algorithm!

Algorithm Structures

Iterative (Loops)

Repeat steps until a condition is met.

While apples > 0:

  Eat 1 apple

  apples = apples - 1

Recursive

Function calls itself! Like Russian dolls 🪆

factorial(5) calls

factorial(4) calls

factorial(3)... etc

Let's see an algorithm in ACTION! Bubble sort compares neighboring elements and swaps them if needed.

Algorithm Efficiency (Big O)

Some algorithms are FAST 🏎️, others are SLOW 🐢. We measure this with Big O notation.

Big ONameSpeedExample
O(1)Constant🚀 Super fastArray access
O(log n)Logarithmic⚡ Very fastBinary search
O(n)Linear🏃 FastFinding item in list
O(n²)Quadratic🚶 SlowBubble sort
O(2ⁿ)Exponential🐢 Very slowBrute force passwords

Summary

✅ Algorithm = ordered, clear, terminating steps to solve a problem

✅ Structures: Iterative (loops) and Recursive (self-calling)

✅ Big O measures algorithm speed

✅ O(log n) < O(n) < O(n²) — smaller is faster
